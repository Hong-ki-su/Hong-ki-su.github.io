[웹 모의해킹] 인프라 보안 수업 - 스캐닝 도구 활용 및 입력값 검증 우회 실무

날짜: 2026년 7월 9일
주제: Kali Linux, Nmap, Nikto, Dirb, 파일 업로드 표적 식별, 입력값 우회 테스트

📝 오늘 배운 내용 상세 요약

1. 정보 수집(Reconnaissance) 및 스캐닝 도구 활용

  • Nmap (포트 및 서비스 스캔): 타겟 서버(192.168.55.111)를 스캔하여 80번 포트(HTTP)가 오픈되어 있음을 확인하고, 구동 중인 환경(Apache 2.4.16, PHP 5.5.13)의 상세 버전을 식별했습니다.
  • Nikto (웹 서버 취약점 스캔): 서버에 설치된 소프트웨어들이 EOL(End of Life, 지원 종료)된 구버전임을 확인하고, CSP, X-Frame-Options 등 주요 보안 헤더가 누락되어 있는 보안 설정 미흡(Security Misconfiguration) 상태를 파악했습니다.
  • Dirb (디렉터리 브루트포싱): 웹 서버 내 숨겨진 경로나 파일을 탐색하여 /phpinfo 경로를 발견(접근 시도 결과 403 Forbidden 상태 확인)했습니다.

2. 웹 애플리케이션 주요 공격 표적(Attack Surface) 식별

  • 파일 업로드 기능 발견: gmeditor/media.php 경로에서 파일을 업로드할 수 있는 폼(Form)을 확인하여, 향후 악성 스크립트(웹쉘) 업로드 공격의 주요 타겟으로 식별했습니다.
  • WYSIWYG 에디터 발견: gmeditor/demo.php 경로에서 웹 에디터를 확인하여, 악성 자바스크립트 삽입을 통한 XSS(Cross-Site Scripting) 취약점 존재 가능성을 파악했습니다.

3. 입력값 검증 우회 기법 테스트

  • URL 파라미터 조작 시도: URL 경로에 %0A (Line Feed, 줄바꿈 문자)를 인코딩하여 삽입(.../java%0Ascript...)함으로써, 웹 서버의 필터링 로직을 우회할 수 있는지 블랙박스 테스트를 진행함(404 Not Found 에러 확인).

💭 오늘의 회고 및 실무 다짐

📌 배운 점

  • 가상 환경을 직접 구축하고, ’정찰 → 서비스 식별 → 공격 표적 발견’으로 이어지는 모의해킹의 정석적인 프로세스를 몸소 체험하며 전체적인 흐름을 이해할 수 있었습니다.
  • 단순한 툴 사용을 넘어, Nmap과 Nikto의 출력 로그에서 타겟의 핵심 정보(서버 버전, 보안 취약점)를 읽어내고 해석하는 방법을 배웠습니다.

🚨 어려운 점 및 개선할 점

  • 스캐닝 도구가 쏟아내는 방대한 결과 로그 중에서, 실제 공격에 유효한 ’크리티컬(Critical)’한 정보를 선별해 내는 직관을 더 길러야겠다고 느꼈습니다.
  • 페이로드 조작(예: %0A 삽입) 시 서버가 반환하는 에러 코드(404 등)의 의미를 분석하여, 필터링 로직이 어떻게 구성되어 있는지 역추적하는 과정이 까다로웠습니다.

💡 실무를 위한 액션 플랜

  • 오늘 식별한 파일 업로드 폼(gmeditor/media.php)을 타겟으로 삼아, 단순 php 확장자 차단을 우회할 수 있는 다양한 기법(예: .php5, .phtml, 대소문자 섞기 등)을 리스트업하고 내일 직접 테스트해 볼 예정입니다.
  • 에디터 페이지 취약점 점검을 위해, 가장 기복 있고 확실한 XSS 확인용 페이로드(예: <script>alert(1)</script>) 5가지를 미리 정리해 두겠습니다.

🤝 함께 나누고 싶은 점

  • Nikto 스캔 결과를 보면서, 지원이 종료된 구버전(PHP 5.5, Apache 2.4.16)을 사용하는 것이 얼마나 많은 보안 위협을 초래하는지 직접 체감했습니다. 시스템 업데이트와 보안 패치의 중요성을 다시 한번 깨닫는 계기가 되었습니다.
