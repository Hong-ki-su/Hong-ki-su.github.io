## 📝 오늘 배운 내용 상세 요약

### 1. 클라이언트 및 서버 사이드 웹 취약점 분석 (Webhacking.kr)
* **JS 로직 우회 (js-2, js-4):** 클라이언트 측 자바스크립트 코드를 분석하여 특정 연산 값(`7809297.1`)이나 조건을 강제로 우회하여 Flag를 획득하는 기법을 실습했습니다.
* **Base64 파일 다운로드 취약점 (web-20):** `?down=` 파라미터에 입력되는 값이 Base64로 인코딩됨을 확인하고, 이를 조작하여 서버 내 임의의 파일을 다운로드하는 취약점을 공략했습니다.
  * 인코딩 테스트: `sudo echo "test.txt" | base64` (출력: `dGVzdC50eHQK`)
  * 디코딩 테스트: `sudo echo "dGVzdC50eHQK" | base64 -d` (출력: `test.txt`)
* **웹쉘 업로드 (포트 10004):** 파일 업로드 폼을 통해 시스템 내부 파일(`/flag`)을 읽어내는 RCE(원격명령실행) 공격 환경을 구축하고 실습을 진행했습니다.

### 2. HTTP 네트워크 통신 및 타겟 서버(192.168.55.120) 구조 파악
* **명령어:** `sudo curl http://192.168.55.120/ -v`
* **분석 결과 1 (공격 벡터 식별):** 응답된 HTML 주석 및 폼 구조를 분석한 결과, 해당 서버의 주요 공격 벡터가 파일 업로드 폼(`<form action="upload.php"...>`)을 통한 RCE(웹쉘)와 HTML 삽입(XSS)임을 파악했습니다.
* **분석 결과 2 (LFI 가능성 확인):** 서버 응답 헤더에서 `Set-Cookie: infofile=imginfo%2F...` 값을 확인하였으며, 추후 해당 쿠키 값 변조를 통한 LFI(Local File Inclusion) 공격 가능성을 식별했습니다.
