# [Ubuntu/Monitoring] 인프라 보안 수업 - Docker 기반 Zabbix 및 PMM 모니터링 시스템 구축

**날짜:** 2026년 4월 28일  
**주제:** Ubuntu 24.04 LTS, APM 스택, Docker, 인프라 관제(Zabbix, PMM)

---

## 📝 오늘 배운 내용 요약

### 1. 서버 및 모니터링 기본 환경 구축
* **인프라 세팅:** 우분투(Ubuntu 24.04 LTS) 기반으로 서버 및 클라이언트 가상머신 환경을 세팅했습니다.
* **APM 스택 구축:** Apache, PHP, MariaDB 패키지 설치를 통해 웹 및 데이터베이스 서버 인프라의 뼈대를 구성했습니다.

### 2. Docker 기반 PMM(Percona Monitoring and Management) 연동
* **Docker 설치:** GPG 키 등록 및 공식 저장소(Repository)를 활용하여 Docker를 안전하게 설치하고 구동 테스트를 완료했습니다.
* **PMM 환경 구성:** PMM 서버 컨테이너를 배포하고 클라이언트(`pmm-client`)와 성공적으로 연동했습니다.
* **권한 제어:** MariaDB 내에 모니터링 전용 계정(`pmm`)을 생성하고 데이터를 수집할 수 있도록 적절한 권한을 부여했습니다.

### 3. Zabbix 관제 시스템 구축 및 DB 설정
* **Zabbix DB 연동:** Zabbix 운영을 위한 전용 데이터베이스(`my_zabbix_db`)와 사용자(`zabbix_user`)를 생성했습니다.
* **UI 구성:** Zabbix 웹 UI 초기 Pre-installation(설전 구성)을 완료하고 DB 커넥션 테스트를 성공적으로 마쳤습니다.

---

## 💭 오늘의 회고

### 배운 점
* 리눅스 환경에서 단순 설치를 넘어, 공식 Repository를 추가하고 GPG 키를 관리하여 안전하게 패키지를 설치하는 실무적인 흐름을 익혔습니다.
* 인프라 모니터링 시스템의 핵심인 PMM과 Zabbix의 아키텍처를 이해하고, 서버-클라이언트 구조로 직접 연동해 보며 전체적인 데이터 수집 흐름을 파악했습니다.
* 외부 모니터링 시스템이 데이터를 수집할 수 있도록 DB(MariaDB)에서 `grant` 구문을 사용하여 적절한 권한을 세밀하게 제어하는 방법을 실습했습니다.

### 트러블슈팅 (어려운 점 및 극복 과정)
* **초기 뼈아픈 실수:** 실습 초반에 우분투 가상머신의 관리자 비밀번호가 기억나지 않아 접속에 계속 실패했고, 결국 OS를 다시 재설치해야 했습니다. 꽤 번거롭고 체력적으로 힘들었지만, 덕분에 초기 세팅을 처음부터 복습하며 인프라 구축 속도를 높일 수 있었습니다.
* **SQL 문법 오류:** Zabbix DB 권한 부여 과정 중 SQL 문법 오류(ERROR 1064)가 발생했으나, 당황하지 않고 쿼리의 오타와 구문을 꼼꼼히 확인하고 수정하여 무사히 해결했습니다.

### 💡 액션 플랜 (네트워크 관제 실무 준비)
* **스냅샷 습관화:** 가상머신 비밀번호나 주요 계정 정보는 확실히 메모해 두고, 중요한 실습 전에는 가상머신 **스냅샷(Snapshot)**을 틈틈이 찍어두어 롤백(Rollback)에 대비하는 엔지니어의 기본 습관을 들이겠습니다.
* **실전 관제 훈련:** 단순히 설치에서 끝내지 않고, 목표로 하는 **네트워크 관제 실무**에 직접적으로 쓰일 수 있도록 Zabbix와 PMM UI에 접속하여 실전처럼 알람(Alert) 룰을 어떻게 세밀하게 조정하는지 직접 만져보고 익히겠습니다.

### 🤝 함께 나누고 싶은 점
* 우분투 기본 저장소에서 `docker-ce` 패키지를 찾지 못해 설치가 막혔을 때, 도커 공식 GPG 키를 추가해서 해결하는 트러블슈팅 과정을 동기들과 공유하여 집단 지성을 높이겠습니다.
