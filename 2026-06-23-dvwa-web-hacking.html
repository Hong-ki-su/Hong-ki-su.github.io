[웹 모의해킹] 인프라 보안 수업 - DVWA 기반 웹 애플리케이션 취약점 분석 및 공격 실무

날짜: 2026년 6월 23일
주제: Kali Linux, Slitaz, Hydra, OWASP ZAP, DVWA (File Upload, XSS)

📝 오늘 배운 내용 상세 요약

1. 타겟 웹 서버(가상머신) 구축 및 네트워크 인프라 설정

  • 운영체제 세팅: 초경량 리눅스 배포판인 Linux Slitaz를 타겟 웹 서버 운영체제로 구축했습니다.
  • 수동 IP 할당 (Static Routing): DHCP(자동 할당)에 의존하지 않고, /etc/network.conf 파일을 vi 편집기로 직접 수정하여 서버망에 맞는 수동 IP를 할당하는 인프라 네트워크 기초 실습을 진행했습니다.
  • 설정 적용 및 검증: /etc/init.d/network.sh restart 명령어를 통해 네트워크 데몬을 재시작하고, eth0 인터페이스에 고정 IP(192.168.55.77, Subnet: 255.255.0.0)가 성공적으로 할당되었음을 ifconfig로 최종 확인했습니다.

2. 웹 취약점 분석 도구(툴) 활용 실무

  • Hydra를 이용한 Brute Force 공격: 타겟 서버(192.168.55.77)의 로그인 폼(http-get-form)을 대상으로 admin 계정의 비밀번호 무차별 대입 공격을 수행하여, 실제 유효한 패스워드(password)를 탈취하는 데 성공했습니다.
  • OWASP ZAP (웹 프록시) 연동: 웹 브라우저(Firefox)와 타겟 서버 간에 오가는 통신(HTTP 트래픽)을 중간에서 가로채기(Intercept) 위한 프록시 환경을 설정하고, 내장 브라우저(Manual Explore)를 활용해 웹 패킷의 구조를 분석하는 방법을 학습했습니다.

3. DVWA를 활용한 웹 애플리케이션 취약점(OWASP) 실습

  • 보안 레벨 테스트: DVWA Security 설정을 Low, Medium, High, Impossible 4단계로 변경해가며, 개발자의 시큐어 코딩(필터링) 수준에 따라 공격 방어 로직이 어떻게 달라지는지 비교 분석했습니다.
  • File Upload (웹쉘 업로드) 취약점:
    • 악의적인 시스템 명령어(system('uname -a; id');)가 포함된 PHP 웹쉘(cmd.php) 파일을 직접 제작하여 서버에 업로드했습니다.
    • 업로드된 경로(/hackable/uploads/cmd.php)로 직접 URL을 타고 접속하여, 타겟 서버의 커널 정보(Linux slitaz) 및 웹 서버 구동 권한(uid=80(www))을 성공적으로 탈취했습니다.
  • XSS (Cross-Site Scripting) 취약점:
    • Reflected XSS: URL 파라미터에 악성 자바스크립트를 주입하여, 링크를 클릭한 희생자 브라우저에서 세션 쿠키(PHPSESSID)가 탈취될 수 있음을 증명했습니다.
    • Stored XSS: 게시판(Guestbook) 입력창에 <script>alert("hack")</script>를 삽입하여 DB에 악성 스크립트를 영구적으로 저장시키고, 해당 페이지에 접속하는 모든 사용자의 브라우저에서 스크립트가 강제 실행되도록 공격을 구현했습니다.

💭 오늘의 회고 및 실무 다짐

📌 배운 점

  • 단순히 웹 해킹 툴만 사용하는 것을 넘어, 인프라 단의 네트워크 설정(IP 할당, 통신 확인)부터 애플리케이션 단의 취약점 공격(웹쉘, XSS)까지 이어지는 풀스택(Full-stack) 모의해킹의 전체적인 흐름을 이해할 수 있었습니다.
  • DVWA의 보안 레벨(Security Level)을 직접 변경해보며, 개발자가 사용자 입력값을 어떻게 필터링(검증)하느냐에 따라 시스템 장악 여부가 완전히 달라진다는 시큐어 코딩의 중요성을 체감했습니다.

🚨 트러블슈팅 (어려운 점 및 개선할 점)

  • 프록시 설정 혼선: OWASP ZAP을 처음 설정할 때, 타겟 IP(192.168.55.77)로 통신하는 기록이 프록시에 전혀 잡히지 않아 공격 시도조차 못 하는 문제가 있었습니다.
  • 해결 및 깨달음: 원인을 분석해보니 브라우저 자체의 ‘로컬망(Localhost/192.168.x.x) 프록시 예외 처리’ 설정이 켜져 있어서 트래픽이 우회된 것이었습니다. 해킹 툴을 사용하기 전에는 반드시 네트워크 연결 상태(ping)와 브라우저 프록시 환경 설정을 꼼꼼히 점검해야 함을 뼈저리게 느꼈습니다.

💡 실무를 위한 액션 플랜

  • 오늘 실습한 Low 레벨의 기초적인 공격 페이로드(PHP 웹쉘 코드, XSS 자바스크립트 등)를 나만의 해킹 치트시트(Cheat Sheet)에 잘 정리해 두겠습니다.
  • Medium이나 High 레벨에서는 오늘 사용한 페이로드가 어떻게 막히는지 서버 측 소스코드를 직접 확인해보고, 이를 우회(Bypass)하는 심화 기법(대소문자 혼용, 널 바이트 삽입 등)을 추가로 찾아 학습하겠습니다.
