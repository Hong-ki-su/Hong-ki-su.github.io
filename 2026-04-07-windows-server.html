[Windows Server/Linux] 인프라 보안 수업 - 계정 보안 및 IIS 웹 서버 구축

날짜: 2026년 4월 7일
주제: 리눅스 계정 관리 심화, Windows Server 2022 설치, IIS(Web Server) 구축

📝 오늘 배운 내용 요약

1. 리눅스 시스템 관리 및 계정 보안

  • 계정 관리 파일 분석: /etc/passwd(사용자 정보), /etc/shadow(비밀번호 보안), /etc/group(그룹 정보)의 구조와 각 필드의 의미를 실습하며 시스템 보안의 기초를 다졌습니다.
  • 명령어 심화 활용: ls 옵션 조합(-lhs, -ltr), 문자열 검색(grep), 명령어 별칭 설정(alias) 및 도움말 확인(man, whatis) 등 효율적인 시스템 관리를 위한 명령어 팁을 익혔습니다.

2. Windows Server 2022 설치 및 IIS 웹 서버 구축

  • OS 설치: VirtualBox 가상 환경에서 Windows Server 2022 Standard (Desktop Experience) 버전을 Custom 설치 방식으로 파티션을 직접 구성하여 설치했습니다.
  • IIS 웹 서버 구축:
    • ’서버 관리자’를 통해 Web Server (IIS) 역할을 추가하고 GameDev라는 이름의 신규 웹 사이트를 생성했습니다.
    • 물리적 경로를 C:\inetpub\wwwroot로 지정하여 웹 콘텐츠를 연결했습니다. (FTP의 경우 ftproot 활용)
    • 해당 경로에 index.html 파일을 생성하여 브라우저에서 서버 IP(192.168.55.219)로 접속 시 정상 동작함을 확인했습니다.

💭 오늘의 회고

배운 점

  • 가상머신 특수 키 입력 해결: Windows Server 로그인 시 필수적인 Ctrl + Alt + Del 입력이 호스트 PC의 단축키와 겹쳐 작동하지 않을 때, 당황하지 않고 가상머신 상단 메뉴(입력 → 키보드 → Ctrl-Alt-Del 누르기)를 통해 우회하여 해결하는 방법을 확실히 체득했습니다.
  • 서버 보안 정책 체감: 서버용 운영체제는 초기 설정부터 대소문자, 숫자, 특수기호를 모두 조합해야만 비밀번호 설정이 가능하다는 점을 통해 인프라 보안의 중요성을 실감했습니다.

어려운 점 및 개선할 점

  • 디렉토리 경로 숙지: 윈도우 서버의 C:\inetpub\wwwroot 등 파일 시스템 구조가 복잡하여 원하는 설정 파일을 한눈에 찾기가 쉽지 않았습니다.
  • 보안과 관리의 딜레마: 강력한 보안 정책 때문에 비밀번호가 너무 복잡해져 자칫하면 잊어버릴 위험이 크다고 느꼈습니다. 서버 관리자로서 강력한 비밀번호를 사용하되, 이를 안전하게 기록하고 관리하는 체계적인 습관화가 반드시 필요하다는 것을 깨달았습니다.

💡 액션 플랜

  • C:\inetpub\wwwroot, /var/log 등 두 운영체제에서 자주 사용하는 핵심 경로들을 리스트로 만들어 눈에 완벽히 익히고 접근 속도를 높이겠습니다.
