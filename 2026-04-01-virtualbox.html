[VirtualBox/Linux] 인프라 보안 수업 - Apache 웹 서버 및 MariaDB 구축

날짜: 2026년 4월 1일
주제: VirtualBox, 리눅스 서버 설정(Apache, MariaDB)

📝 오늘 배운 내용 요약

1. Apache (httpd) 웹 서버 설정

  • 패키지 설치 및 실행: dnf install httpd 명령어를 통해 Apache 웹 서버를 설치하고, systemctl start httpd로 서비스를 실행했습니다.
  • vi 에디터 활용 팁: vi 편집기에서 명령 모드(Esc)로 진입한 후 :set nu 또는 :set number를 입력하면 화면에 줄 번호가 표시됩니다. 이를 통해 설정 파일 내 특정 행(예: 포트를 설정하는 47행의 Listen 80)을 찾아 수정하는 작업이 훨씬 수월해졌습니다.

2. MariaDB 데이터베이스 (SQL) 조작

  • 테이블 구조 변경: alter table student drop [컬럼명] 명령어를 통해 테이블에서 불필요한 열을 삭제하는 방법을 익혔습니다.
  • 데이터 관리 (CRUD):
    • Insert: insert into student values (...) 명령어로 새로운 학생 데이터 추가
    • Select: select * from student 명령어로 저장된 전체 데이터 조회
    • Update: update student set ... where ... 조건을 활용해 특정 데이터 내용 수정
    • Delete: delete from student where ... 조건을 이용해 특정 행 삭제

💭 오늘의 회고

배운 점

  • 순수 리눅스 터미널 환경에서 기본 네트워크를 설정하고, 웹(Web) 서버와 DB 서비스를 직접 구축해 보며 전체적인 IT 인프라 흐름을 파악할 수 있었습니다.
  • vi 에디터에서 줄 번호를 띄우는 사소하지만 강력한 기능을 배워, 앞으로 복잡하고 긴 서버 설정 파일을 다룰 때 작업 편의성이 크게 올라갈 것 같습니다.

트러블슈팅 및 개선할 점

  • 네트워크 설정 숙달: 리눅스 네트워크 인터페이스 설정 시 사용하는 nmcli 명령어가 아직은 조금 복잡하게 느껴져 반복적인 실습이 필요합니다.
  • 가상머신 접속 오류 해결: 실습 도중 가상머신 오류와 네트워크 단절 현상이 겹쳐 진행에 큰 어려움을 겪었습니다. 하지만 혼자 끙끙대지 않고 Discord로 즉각적인 질문을 남겼고, 감사하게도 친절한 피드백을 받아 가상머신 및 Putty 원격 접속 네트워크 오류를 무사히 해결했습니다. 문제가 생겼을 때 적극적으로 질문하고 소통하여 해결해 내는 과정의 중요성을 느꼈습니다.
