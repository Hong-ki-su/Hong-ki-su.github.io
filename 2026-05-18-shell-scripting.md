## 📝 오늘 배운 내용 상세 요약

### 1. 쉘 스크립트 (Shell Script) 기초 문법 및 활용
* **변수와 입출력:** `test.sh` 파일에서 환경 변수(`NAME`, `VERSION` 등)를 선언하고 `echo`로 출력하는 법을 익혔으며, `login.sh`에서는 `read` 명령어를 활용해 사용자의 입력을 동적으로 처리하는 방법을 실습했습니다.
* **조건문과 반복문:** `if.sh`를 작성하여 상황에 따른 조건 판단 로직을 구현하고, `for`와 `while` 루프의 개념을 이해하여 반복적인 시스템 작업을 자동화하는 기반을 다졌습니다.
* **파일 및 디렉터리 검증:** `dir.sh` 스크립 내에서 `if [ -d ... ]` 구문을 사용하여 특정 디렉터리의 존재 여부를 선행 확인하고, 없을 경우에만 안전하게 생성(`mkdir`)하는 실무적인 방어 로직을 작성했습니다.

### 2. 리눅스 파일 권한 및 소유권 이해
* **실행 권한 부여:** 리눅스 환경에서 작성한 쉘 스크립트를 실제 데몬이나 프로그램처럼 실행하기 위해 `chmod +x 파일명` 명령어로 실행 권한을 부여하는 필수 과정을 학습했습니다.
* **권한 트러블슈팅:** `root` 관리자가 소유한 중요 파일에 일반 계정으로 권한 수정을 시도할 때 발생하는 `Operation not permitted` 에러의 원리를 이해하고, 이를 `sudo` 권한 위임을 통해 합법적으로 해결하는 방법을 체득했습니다.

### 3. 텍스트 처리 및 데이터 추출 명령어 (awk, sed)
* **awk 텍스트 파싱:** 방대한 텍스트나 로그에서 특정 열을 추출(`print $1, $3`)하고, 콜론 등 구분자를 지정(`-F:`)하며, 특정 조건에 맞는 데이터만 필터링(`$3 >= 30`)하는 강력한 데이터 가공 실습을 진행했습니다.
* **sed 스트림 편집:** 텍스트 내의 특정 패턴을 일괄 치환(`s/status=disable/status=enable/g`)하거나 불필요한 라인을 삭제(`2d`)하고, `-i` 옵션을 통해 원본 파일을 스크립트 상에서 직접 수정하는 실무 기법을 익혔습니다.

### 4. 시스템 및 네트워크 모니터링 스크립트 실전 작성
* **디스크 모니터링 (`disk.sh`):** `df -h` 명령어의 출력 결과를 `awk`와 `sed`로 파이프라이닝하여 현재 디스크 사용량(%) 수치만 정확히 추출해 내고, 이 값이 80%를 초과할 경우 관리자에게 경고 메시지를 띄우는 실전 자동화 로직을 구현했습니다.
* **IP 주소 추출 (`ip.sh`):** `ip addr` 명령어의 복잡한 네트워크 인터페이스 결과값 속에서, 파싱을 통해 현재 시스템에 할당된 유효 IP 주소만 깔끔하게 추출해 내는 스크립트를 작성했습니다.

### 5. 패키지 관리 및 로그 분석 도구 설치
* **APT 저장소 트러블슈팅:** `apt` 패키지 저장소 리스트를 수동으로 추가하는 과정에서 발생한 구문 오류(`Malformed entry`)를 로그를 통해 직접 디버깅하여 해결했습니다.
* **GoAccess 로그 시각화:** 실시간 웹 로그 분석기인 `GoAccess`를 설치하고, Apache 웹 서버의 원시 로그 파일(`access.log`)을 시각적인 HTML 리포트 포맷으로 변환하여 분석 및 출력하는 실습을 완료했습니다.

## 💭 오늘의 회고 및 실무 다짐

### 📌 배운 점
* 단일 명령어를 개별적으로 치는 수준을 넘어, 여러 명령어(`df`, `awk`, `sed`)를 파이프라인(`|`)으로 유기적으로 연결하고 이를 쉘 스크립트로 묶어 시스템 관리를 자동화하는 엔지니어링 기법을 직접 구현해 보았습니다.
* 특히 앞으로 목표로 하는 **인프라 보안 및 네트워크 모니터링(NOC)** 업무에서, 정규화되지 않은 방대한 텍스트 로그를 스크립트를 통해 빠르고 정확하게 분석하고 서버 상태를 점검하는 능력이 얼마나 중요한지 깊이 체감했습니다.

### 🚨 트러블슈팅 (어려운 점 및 개선할 점)
* **정규표현식의 엄격함:** `awk`와 `sed` 명령어 사용 시 정규표현식 문법이나 기호(따옴표, 슬래시 등)가 단 하나라도 틀리면 스크립트가 완전히 오작동하기 때문에 초기 작성이 매우 까다롭게 느껴졌습니다. 쉘 스크립팅은 정확한 타이핑과 문법 숙지가 생명임을 배웠습니다.
* **설정 파일 구문 오류:** APT 저장소 설정 파일을 수정할 때 닫는 괄호(`]`) 하나가 빠지는 사소한 실수로 인해 시스템의 전체 패키지 업데이트가 막히는(`Malformed entry`) 아찔한 경험을 했습니다. 서버의 핵심 설정 파일을 건드릴 때는 토씨 하나까지 꼼꼼히 점검해야 함을 깨달았습니다.
