[VirtualBox/Linux] 인프라 보안 수업 - Rocky Linux 서버 인프라 구축 및 보안 설정

날짜: 2026년 4월 2일
주제: VirtualBox, Rocky Linux, 웹/DB/DNS 서버 구축, 방화벽(FirewallD) 관리

📝 오늘 배운 내용 요약

1. Rocky Linux 기반 네트워크 및 서비스 인프라 구축

  • 초기 설정: 기존 VM 삭제 후 Rocky Linux를 재설치하고, ip addr 명령어를 통해 네트워크 인터페이스의 정상 할당 상태를 확인했습니다.
  • 패키지 설치: dnf 패키지 관리자를 활용하여 웹(httpd), DB(mariadb), DNS(bind) 등 서버 운영에 필수적인 핵심 서비스 패키지들을 일괄 설치했습니다.

2. 웹 서비스(HTTP/HTTPS) 구축 및 방화벽(FirewallD) 보안 설정

  • 웹 서버 및 암호화 통신: Apache 웹 서버를 가동하여 초기 자기소개 페이지(index.html)를 생성하고, mod_ssl 패키지 설치를 통해 HTTPS(443 포트)를 활성화하여 SSL 보안 통신 환경을 구성했습니다.
  • 방화벽 정책 관리 (firewalld):
    • firewall-cmd 명령어를 활용하여 DNS, HTTP, HTTPS, FTP 등 서비스 운영에 필수적인 포트들의 접근을 허용하도록 방화벽 정책을 설정했습니다.
    • FTP 패시브(Passive) 모드 운영을 위해 특정 데이터 전송 포트 범위(예: 50000-55000)를 추가로 개방하는 실습을 진행했습니다.

💭 오늘의 회고

배운 점

  • 도메인(예: www.test.com)을 통해 사용자가 웹 사이트에 접속하기 위해서는, 단순히 웹 서버만 띄우는 것이 아니라 DNS 서버의 존(Zone) 파일 설정과 웹 서버의 VirtualHost 설정이 유기적으로 연결되어야 함을 배웠습니다. 인프라는 각 서비스 간의 연결고리가 핵심이라는 것을 체감했습니다.

트러블슈팅 및 개선할 점

  • 다각적인 원인 분석: ERR_CONNECTION_REFUSED (연결 거부) 오류가 발생했을 때, 처음에는 단순한 방화벽 문제로만 접근했습니다. 하지만 실제로는 서비스 데몬의 활성화 상태(systemctl status), 포트 리스닝 상태, SSL 모듈 정상 설치 여부 등 다각도로 원인을 파악하고 접근해야만 정확한 해결이 가능함을 느꼈습니다.
  • 설정의 연쇄 작용: 사소한 설정 하나를 놓치면 아무리 DNS 서버를 완벽하게 구축해 두어도 클라이언트가 도메인을 찾지 못해 서비스가 마비된다는 점을 뼈저리게 경험했습니다.
