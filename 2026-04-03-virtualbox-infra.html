[VirtualBox/Linux] 인프라 보안 수업 - 종합 서버 인프라 연동 및 트러블슈팅

날짜: 2026년 4월 3일
주제: VirtualBox, DNS/Web/DB/FTP 서버 종합 연동, SELinux 및 권한 트러블슈팅

📝 오늘 배운 내용 요약

1. 인프라 서버 구축 및 연동 (Server: hong / Client: root)

  • DNS 서버: gisu.com 도메인 설정 및 호스트 레코드(www, ftp)를 등록하여 네임서버를 구축했습니다.
  • Web/DB 서버: Apache SSL(HTTPS) 보안 설정과 더불어 MariaDB 기반의 데이터베이스를 구축하여 웹 서비스 환경을 구성했습니다.
  • FTP 서버: vsftpd 서비스를 구축하고 사용자(hong) 계정을 연동하여 파일 전송 시스템을 마련했습니다.

2. 클라이언트 접속 및 환경 설정

  • 로컬 DNS 오버라이딩: 윈도우 hosts 파일 수정을 통해 로컬 환경에서 DNS를 강제로 매핑하는 기법을 학습했습니다.
  • 접속 테스트: FileZilla를 이용한 SFTP/FTP 프로토콜 접속 테스트를 통해 서버-클라이언트 간의 정상적인 통신을 확인했습니다.

💭 오늘의 회고

배운 점

  • 서버 한 대를 단편적으로 다루는 것을 넘어, DNS-Web-DB-FTP가 서로 어떻게 얽혀서 돌아가는지 전체적인 인프라 아키텍처의 구조를 깊이 있게 이해하게 되었습니다.

트러블슈팅 (어려운 점 및 해결 과정)

  • DNS 인식 문제: 윈도우(호스트)가 가상머신 내부에 구축한 DNS를 무시하고 외부 IP로 연결을 시도하는 현상 때문에 통신 흐름을 잡는 데 많이 헤맸습니다.
  • 권한 및 방화벽 에러: hosts 파일 수정 시 발생하는 윈도우 관리자 권한 문제와, 리눅스 내부의 SELinux 차단 문제로 인해 연결 재설정 오류가 빈번하게 발생했습니다. 여러 권한이 꼬이면서 원인을 찾기 위해 AI의 도움을 받아 디버깅을 진행했으며, 인프라 환경에서는 단순한 오타보다 권한 및 보안 정책(SELinux, Firewall)이 장애의 주된 원인이 됨을 체감했습니다.
