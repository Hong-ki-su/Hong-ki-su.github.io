# [웹 모의해킹] 인프라 보안 수업 - HTTP 헤더 변조, JS 난독화 해제 및 웹쉘 업로드 실무

**주제:** User-Agent 변조, 자바스크립트 난독화(Deobfuscation), File Upload, 모의해킹 방법론

## 📝 오늘 배운 내용 상세 요약

### 1. HTTP 헤더(User-Agent) 변조 및 검증 우회 공격
* **취약점 분석:** 서버가 클라이언트의 브라우저 정보(User-Agent)를 검증하여 특정 대상만 접근을 허용하는 로직의 취약점을 파악했습니다.
* **프록시 변조:** Burp Suite를 활용해 POST 요청을 가로챈 뒤, 기존의 Firefox 브라우저 문자열을 서버가 요구하는 특정 키워드(`SuNiNaTas`)로 위조하여 인증 필터를 우회하는 실습을 수행했습니다.

### 2. 자바스크립트 난독화(Obfuscation) 분석 및 디오퍼스케이션
* **코드 분석:** 웹 애플리케이션 프론트엔드에 적용된 `eval(function(p,a,c,k,e,r)...)` 형태의 팩커(Packer) 난독화 스크립트를 식별했습니다.
* **디오퍼스케이션(Deobfuscation):** 정적 분석 도구(JSNICE) 및 브라우저 개발자 도구를 활용해 난독화된 코드를 사람이 읽을 수 있는 평문 코드로 복원하여 내부 인증 알고리즘과 Key 값을 도출하는 기법을 학습했습니다.

### 3. 신규 가상 환경(acc01) 타겟 서버 구축 및 파일 업로드 취약점 진단
* **네트워크 환경 확장:** 새로운 가상머신 acc01(192.168.55.216)을 구동하고 `ifconfig`로 인터페이스를 확인하여 모의해킹 타겟 시스템을 추가 확보했습니다.
* **공격 표면 분석:** "계정정보 처리가 취약한 누리집" 웹 서비스를 대상으로, 서버가 지원하는 서버 사이드 스크립트 확장자(php, jsp, asp, aspx)와 이중 확장자 우회 흔적(`devshell.php.jpg`)을 분석하며 파일 업로드(File Upload) 취약점 및 웹쉘(WebShell) 침투 경로를 탐색했습니다.

### 4. 모의해킹 실무 방법론 및 인프라 진단 이론
* **수행 프로세스:** 모의해킹 계약 진행 단계부터 '정보 수집/취약점 분석(수동/자동) $\rightarrow$ 모의해킹 수행 $\rightarrow$ 결과 보고서 작성'으로 이어지는 실무 보안 컨설팅 흐름을 정립했습니다.
* **진단 관점 분류:** 대상 시스템 정보 제공 수준에 따른 화이트박스(White), 블랙박스(Black), 그레이박스(Gray) 테스트의 개념과 차이점을 비교 분석했습니다.

🛠 주요 트러블슈팅 (Troubleshooting) 내역
Q. 클라이언트 환경(Browser) 불일치로 인한 인증 실패 에러
원인: 웹 서버가 제공하는 특정 기능이 일반 브라우저 값(Mozilla/5.0...)으로 요청될 경우 접근을 전면 차단하여 ????? 문자열 처리가 되거나 포인트 획득이 거부되었습니다.

해결: 해당 로직이 User-Agent 헤더 값을 기반으로 검증한다는 점을 파악하고, Burp Suite 인터셉트 상태에서 값을 서버 기준인 SuNiNaTas로 변환 및 전송(Forward)하여 우회에 성공했습니다.

Q. 자바스크립트 팩커 난독화(Packer/Eval)로 인한 로직 해석 불가
원인: 가독성이 변형된 스크립트로 인해 클라이언트 측 인증 검증 값이나 힌트를 소스코드 보기 창에서 추출하기 어려웠습니다.

해결: jsnice.org 통계적 리네이밍 및 디오퍼스케이션 엔진을 활용하여, 파라미터 간의 구조를 정렬하고 본래의 리턴 값 구조를 복원하여 코드를 정밀 분석할 수 있었습니다.

💭 오늘의 회고 및 실무 다짐
📌 배운 점
웹 보안은 단순히 서버 측 코드뿐만 아니라 프론트엔드 자바스크립트의 난독화 분석, 클라이언트가 보내는 HTTP 요청 헤더 검증 등 복합적인 요소에 대한 이해가 필수적임을 배웠습니다.

새로운 가상머신 서버를 추가 구축하여 실제 공격 표면(Upload 디렉터리 구조)을 관찰하면서, 인프라 진단 환경을 직접 세팅하는 역량을 기를 수 있었습니다.

🚨 어려운 점 및 개선할 점
난독화된 자바스크립트 코드를 처음 접했을 때 내부 동작을 추적하는 데 어려움이 있었으나, 웹 복원 도구의 도움과 소스코드 가독성 확보(Nicify) 과정을 거치며 우회 포인트를 짚어낼 수 있었습니다.

다중 가상머신 환경을 다룰 때 각 시스템의 IP 대역과 실행 상태를 꼼꼼히 체크하지 않으면 통신 단절이 발생하므로 인프라 맵을 명확히 인지해야 함을 느꼈습니다.

💡 액션 플랜
모의해킹 방법론 단계에 따라, 수동 취약점 분석 시 도구에만 의존하지 않고 패킷의 세부 헤더 값 하나하나를 직접 변조해가며 꼼꼼하게 검증하는 정밀 진단을 습관화하겠습니다.

파일 업로드 취약점 테스트 시 각 웹 서버 환경(Apache, IIS, Tomcat)별로 실행 가능한 확장자 메커니즘을 사전에 완벽히 파악하여 진단 효율을 높이겠습니다.
