[웹 모의해킹] 인프라 보안 수업 - RCE, CSRF 및 접근 통제 결함 취약점 분석

날짜: 2026년 6월 30일 주제: Kali Linux, bee-box, RCE(PHP CGI), CSRF, XXE, SSRF, Broken Access Control

📝 오늘 배운 내용 상세 요약

1. 원격 코드 실행 (RCE) 및 서버 정보 노출

  • PHP CGI 취약점 (CVE-2012-1823 등): phpinfo.php 페이지를 통해 타겟 서버가 취약한 구버전(PHP 5.2.4 CGI/FastCGI)으로 구동 중임을 식별했습니다. 이후 URL 쿼리스트링에 ?-d+auto_prepend_file...와 같은 악의적인 옵션을 삽입하여, 시스템의 핵심 계정 정보가 담긴 /etc/passwd 파일을 웹 화면에 그대로 출력시키는 치명적인 RCE 공격 실습을 진행했습니다.
  • 관리자 페이지 노출 (Misconfiguration): http://192.168.55.150/sqlite/ 경로에 SQLiteManager(버전 1.2.4)가 기본 설정 그대로 외부에 노출되어 있음을 확인했습니다. 이는 인증 없이 데이터베이스 구조를 열람하거나 조작할 수 있는 심각한 설정 오류(Misconfiguration)입니다.

2. 세션 하이재킹 및 위조 공격 (Client-Side Attacks)

  • CSRF (Cross-Site Request Forgery): 게시판 등 취약한 입력 폼에 가로/세로 크기가 0인 보이지 않는 <iframe> 태그를 삽입했습니다. 다른 사용자나 관리자가 해당 게시글을 열람하는 순간, 본인의 의도와 무관하게 특정 계좌로 자금(amount=700)이 이체되도록 유도하는 스크립트 공격 기법을 체득했습니다.
  • User-Agent 분석: 클라이언트가 웹 서버에 접속할 때 남기는 User-Agent 헤더(안드로이드 기기, 브라우저 엔진 정보 등)를 분석했습니다. 이를 변조하여 모바일 환경을 위장하거나 특정 로직을 우회하는 원리를 학습했습니다.

3. 서버 측 요청 위조 및 접근 통제 결함

  • XXE 및 SSRF 연계: Burp Suite의 Intercept 기능을 활용하여 xxe-1.php와 ssrf.php로 향하는 HTTP 패킷을 캡처했습니다. 외부 엔티티나 내부망 시스템으로 비정상적인 요청을 보내도록 유도하는 패킷 조작 과정을 분석했습니다.
  • 접근 통제 우회 (Broken Access Control): ‘Restrict Folder Access’ 실습에서, 정상적으로 로그아웃(Log off)을 한 상태이거나 권한이 없는 사용자임에도 불구하고 디렉토리 리스팅(Directory Listing) 취약점을 통해 내부 PDF 문서 파일들에 접근할 수 있는 결함을 확인했습니다.

💭 오늘의 회고 및 실무 다짐

📌 배운 점

  • 애플리케이션의 설정 오류(PHP CGI)나 단순한 입력값 검증 부재(CSRF 허용) 하나가 시스템 전체의 계정 정보(/etc/passwd) 유출이나 금전적 피해로 이어질 수 있다는 파급력을 두 눈으로 직접 확인했습니다.
  • 단순한 웹 해킹을 넘어 애플리케이션 보안의 중요성을 깊이 체감한 하루였습니다.

🚨 트러블슈팅 (어려운 점 및 개선할 점)

  • CSRF 공격 페이로드를 작성할 때 캡처 화면에 있는 강사님 IP(192.168.56.114)를 무심코 따라 치기 쉬운데, 반드시 내 실습 환경인 192.168.55.150으로 타겟팅을 정확히 맞춰야 스크립트가 성공적으로 동작한다는 점을 다시금 상기했습니다.
  • 스크립트 작성 시 타겟 정보에 대한 꼼꼼한 크로스 체크가 필수적임을 깨달았습니다.

💡 실무를 위한 액션 플랜

  • 네트워크 관제(NOC) 실무 대비: 오늘 실습한 PHP CGI RCE 공격이 발생했을 때, 웹 서버의 액세스 로그(Access Log)나 IDS/IPS 장비에 ?-d+auto_prepend_file와 같은 특이한 페이로드가 어떻게 남는지 분석하고 탐지 룰을 직접 고민해 볼 예정입니다.

🤝 함께 나누고 싶은 점

  • 단순히 웹 취약점을 공격해서 뚫어내는 것에 그치지 않고, “내가 보안 관제 요원이라면 방금 쏜 CSRF나 SSRF 패킷을 와이어샤크나 관제 솔루션에서 어떻게 식별해 낼 수 있을까?”에 대한 역발상적인 고민을 팀원들과 함께 나누어보고 싶습니다.
