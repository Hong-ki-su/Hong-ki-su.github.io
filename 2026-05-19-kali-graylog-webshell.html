[인프라/모의해킹] 인프라 보안 수업 - Graylog 트러블슈팅 및 Kali Linux 웹쉘 침투 실무

날짜: 2026년 5월 19일
주제: 리눅스 트러블슈팅, Graylog, MongoDB, Nmap, Gobuster, Web Shell(웹쉘) 작성

📝 오늘 배운 내용 상세 요약

1. 리눅스 서버 및 Graylog 트러블슈팅 실무

  • 서비스 제어 및 로그 추적: systemctl을 활용해 데몬 상태를 제어(status, stop, start, restart)하고, 서버 데몬이 죽었을 때 tail -n 30 /var/log/... 및 journalctl 명령어를 사용하여 시스템의 핵심 에러 원인(Cause)을 역추적하는 기법을 실습했습니다.
  • 권한 및 DB 꼬임 해결: Graylog 설정 중 발생한 권한 문제(is not writable)를 chown -R graylog:graylog 명령어로 디렉터리 권한을 부여해 해결했습니다. 또한, 프로그램이 무한 재시작될 때 mongosh --eval "db.dropDatabase()" 명령어를 통해 꼬여버린 MongoDB 데이터베이스를 깔끔하게 초기화하는 실무적인 트러블슈팅 기법을 적용했습니다.

2. Nmap을 활용한 네트워크 및 포트 스캔

  • 대역대 식별: nmap -sn 192.168.55.0/24 명령어로 Ping 스캔을 수행하여 네트워크 대역 내에 살아있는 호스트(타겟)를 빠르게 식별했습니다.
  • 상세 서비스 분석: 발견된 타겟 머신(192.168.55.243)에 대해 nmap -sV -p- 명령어를 실행하여, 모든 포트(-p-)를 대상으로 개방된 포트와 구동 중인 서비스의 정확한 버전 정보(SSH, Apache HTTP 등)를 상세히 분석했습니다.

3. Gobuster를 활용한 웹 디렉터리 브루트포싱

  • 숨겨진 단서 탐색: gobuster dir -u [URL] -w [단어장] -x php,txt,bak,html 명령어를 사용하여 웹 서버 내부에 숨겨진 관리자 페이지(대시보드), 백업 파일(.bak), 그리고 특정 동작을 수행하는 스크립트(ajax.php) 등을 브루트포싱 기법으로 샅샅이 탐색했습니다.

4. 웹쉘(Web Shell) 및 리버스 커넥션 준비

  • PHP 백도어 생성: echo '<?php system($_GET["cmd"]); ?>' > shell.php 명령어를 활용하여, URL 파라미터를 통해 서버 시스템 명령어를 원격으로 직접 실행할 수 있는 초간단 PHP 웹쉘(Web Shell)을 작성했습니다.
  • 리스너 오픈: 공격자(Kali) PC에서 nc -lvnp 5555 명령어를 통해 타겟 서버로부터 들어올 리버스 쉘(Reverse Shell) 접속을 대기하는 포트를 개방했습니다.

💭 오늘의 회고 및 실무 다짐

📌 배운 점

  • 서버 에러 발생 시 무작정 설정 파일부터 건드리는 것이 아니라, 로그(Log)에서 Exception, Error, Failed 등의 핵심 키워드를 찾아 정확한 원인(오타, 중복 기입, 권한 문제 등)을 파악하는 것이 엔지니어의 최우선 과제임을 뼈저리게 깨달았습니다.
  • Nmap으로 타겟을 찾고, Gobuster로 웹 페이지의 숨겨진 백도어나 단서를 찾는 일련의 모의해킹 정보 수집(Reconnaissance) 프로세스를 직접 실습하며 공격의 청사진을 그리는 법을 배웠습니다.
  • 웹 서버의 취약점을 공략하기 위해 PHP 웹쉘을 직접 작성해 보고, 이 작은 코드가 서버를 어떻게 장악하는지 그 근본적인 원리를 이해했습니다.

🚨 트러블슈팅 (어려운 점 및 개선할 점)

  • 인프라 설정 장애: Graylog 설정 과정에서 권한 부족(is not writable) 에러나 설정 파일의 중복 기입 문제로 서버가 켜지지 않아 원인을 찾는 데 오랜 시간을 허비했습니다.
  • 툴 사용 미숙: 칼리 리눅스에서 툴 사용 시 절대 경로 오타나 파이썬 버전별 문법 차이(print 함수 괄호 유무 등) 같은 사소한 에러에 대처하는 속도가 아직 미숙함을 느꼈습니다.

💡 실무를 위한 액션 플랜

  • 에러가 났을 때 당황하지 말고, 에러 로그의 가장 마지막 줄(또는 Caused by: 구문)을 꼼꼼히 읽는 침착한 습관을 들이겠습니다.
  • 모의해킹 시 매번 찾아보기 번거로운 자주 쓰는 명령어(Nmap, Gobuster 세부 옵션)와 주요 단어장의 절대 경로를 메모장에 단축키처럼 완벽히 정리해 두겠습니다.

🤝 함께 나누고 싶은 점

  • 트러블슈팅 콤보 공유: 우분투 서버 세팅 중 프로그램이 무한 재시작되며 죽을 때, ‘설정 파일 백업본 복구 + chown 권한 재부여 + DB 찌꺼기 삭제(dropDatabase)’ 콤보로 깔끔하게 해결이 가능합니다! 비슷한 에러를 겪는 팀원들에게 적극적으로 공유하겠습니다.
  • 심화 학습 제안: 오늘 만든 기본 웹쉘을 업로드할 때, 서버 측의 필터링을 우회하는 방법(확장자 변조, HTTP Content-Type 변경 등)에 대해 팀원들과 같이 심도 있게 테스트해 보면 좋을 것 같습니다.
