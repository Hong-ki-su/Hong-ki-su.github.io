# [보안 관제/SIEM] 인프라 보안 수업 - Wazuh 오픈소스 HIDS/SIEM 올인원(All-in-one) 구축 실무

**날짜:** 2026년 7월 20일  
**주제:** Ubuntu, Wazuh, SIEM, HIDS, 자동화 설치 스크립트, 보안 관제 인프라

---

## 📝 오늘 배운 내용 상세 요약

### 1. Ubuntu 리눅스 시스템 패키지 관리 및 최신화
* **패키지 목록 동기화 및 업그레이드:** `sudo apt update -y` 및 `sudo apt upgrade -y` 명령어를 통해 OS 패키지 목록을 동기화하고 최신 버전으로 업그레이드하여 보안 및 안정성을 확보했습니다.
* **네트워크 도구 확인:** 네트워크 테스트 및 파일 다운로드를 위해 `iputils-ping`, `curl` 패키지 설치를 확인했습니다.
* **외부망 통신 점검:** `ping 8.8.8.8` (구글 DNS)을 통해 외부망과의 정상적인 통신 상태 점검을 완료했습니다.

### 2. Wazuh(와주) 올인원(All-in-one) 설치
* **공식 설치 스크립트 다운로드:** `curl`을 사용하여 Wazuh 공식 설치 셸 스크립트(`wazuh-install.sh`)를 다운로드했습니다.
* **자동화 설치 실행:** `sudo bash ./wazuh-install.sh -a` 명령어(all-in-one 옵션)를 사용하여 복잡한 보안 관제 시스템을 자동화된 방식으로 단일 서버에 설치했습니다.

### 3. Wazuh 주요 컴포넌트 설치 및 구동 확인 (로그 분석)
* **Wazuh Indexer:** 보안 데이터를 수집하고 분석 및 검색할 수 있도록 인덱싱하는 엔진을 설치했습니다.
* **Wazuh Manager (Server):** 에이전트로부터 데이터를 수신하고 룰셋에 따라 위협을 탐지하는 핵심 서버를 구동했습니다.
* **Filebeat:** Manager가 수집한 경고 및 이벤트를 Indexer로 안전하게 전달하는 데이터 수집기 설정을 완료했습니다.
* **Wazuh Dashboard:** 수집된 보안 데이터를 시각화하여 관제할 수 있는 웹 UI를 설치 및 443(HTTPS) 포트를 개방했습니다.
* **관리자 계정 발급:** Dashboard 접속을 위한 `admin` 계정과 초기 비밀번호를 확인했습니다.

## 💭 오늘의 회고 및 실무 다짐

### 📌 배운 점
* 단순한 리눅스 명령어 실습을 넘어, 실제 NOC(네트워크 관제 센터)나 SOC(보안 관제 센터) 환경에서 사용되는 오픈소스 HIDS/SIEM 솔루션의 인프라 구축 과정을 경험했습니다.
* 자동화 스크립트가 내부적으로 인증서 생성, 컴포넌트 간 연동(Manager -> Filebeat -> Indexer), 웹 대시보드 기동 등을 순차적으로 처리하는 흐름을 파악했습니다.

### 🚨 어려운 점 및 개선할 점
* `-a` 옵션을 통해 원클릭으로 쉽게 설치했지만, 내부적으로 각 컴포넌트가 어떤 포트로 어떻게 통신하는지(예: 1514, 9200 등) 세부적인 구조를 완벽히 이해하기에는 추가적인 학습이 필요합니다.

### 💡 실무를 위한 액션 플랜
* 발급받은 IP와 비밀번호로 Wazuh Dashboard 웹 UI에 접속하여 정상 동작 여부를 확인하겠습니다.
* 초기 비밀번호는 보안상 취약할 수 있으므로, 대시보드 접속 후 관리자 비밀번호 변경 또는 안전한 곳에 기록해 두겠습니다.
* `/var/log/wazuh-install.log` 파일을 확인하여 설치 중 발생한 세부 로그를 가볍게 훑어보겠습니다.

### 🤝 함께 나누고 싶은 점
* 셸 스크립트를 활용한 인프라 자동 구축의 편리함을 느꼈습니다. 시스템 엔지니어가 수동으로 구성하면 반나절 이상 걸릴 작업을 스크립트 하나로 몇 분 만에 끝낼 수 있다는 점이 매우 인상적입니다.
