# [CISCO] 인프라 보안 수업 - DHCP 서버 및 L3 스위치 라우팅 설정

**날짜:** 2026년 3월 27일  
**주제:** CISCO, DHCP, L3 스위치

---

## 📝 오늘 배운 내용 요약

### 1. Cisco Router를 이용한 DHCP 서버 설정
* **`ip dhcp pool`**: 주소 할당을 위한 풀(Pool) 생성
* **`network [네트워크 주소] [서브넷 마스크]`**: 할당할 IP 대역 지정
* **`default-router`, `dns-server`**: 클라이언트에게 전달할 디폴트 게이트웨이 및 DNS 정보 설정
* **`ip dhcp excluded-address`**: 특정 범위의 IP가 자동 할당되지 않도록 제외 (IP 충돌 방지 및 관리용 목적)

### 2. L3 스위치 인터페이스 및 라우팅 설정
* **`no switchport`**: L2 포트의 특성을 L3 포트로 전환하여 스위치 포트에 직접 IP 할당이 가능하도록 설정
* **`ip routing`**: L3 스위치 내부에서 라우팅 기능을 활성화 (EIGRP 등 라우팅 프로토콜을 사용하기 위한 필수 전제 조건)
* **`show ip dhcp binding`**: 현재 어떤 IP 주소가 어떤 기기의 MAC 주소에 할당되었는지 상태 확인

---

## 💭 오늘의 회고

### 배운 점
* 라우터에서 DHCP를 설정할 때 단순히 IP 대역만 정하는 것이 아니라, 제외 주소(`excluded-address`)를 설정하여 IP 충돌을 사전에 방지하는 실무적인 방법을 배웠습니다.
* L3 스위치 포트에 IP가 입력되지 않을 때, `no switchport` 명령어로 포트의 성격을 먼저 바꿔주어야 한다는 점을 확실히 인지하게 되었습니다.

### 어려운 점 및 개선할 점
* DHCP 설정 모드(`dhcp-config`) 내에서 오타가 나거나 잘못된 명령어를 입력했을 때 순간적으로 당황했습니다. 앞으로는 CLI 환경에서 `?`(도움말) 기능을 적극적으로 활용하여 정확한 명령어를 스스로 찾는 연습을 더 해야겠습니다.
* 금요일에 부여받은 라우터 관련 실습 과제를 수행하며 헷갈리는 부분이 많아 어려움을 겪었습니다. 하지만 주말 동안 포기하지 않고 끈질기게 원인을 찾아 해결하여 무사히 과제를 제출했습니다. 네트워크 인프라는 반복적인 실습과 복습만이 답이라는 것을 다시 한번 느꼈습니다.o
