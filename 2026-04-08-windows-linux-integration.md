# [통합 인프라] 인프라 보안 수업 - 이기종 OS 환경 통합 및 중앙 집중형 로그 서버 구축

**날짜:** 2026년 4월 8일  
**주제:** Windows Server 2022 & Rocky Linux 통합 인프라 연동, rsyslog 중앙 집중화

---

## 📝 오늘 배운 내용 요약

### 1. 중앙 집중형 로그 서버 구축 및 데이터베이스 연동
* **로그 자동화 수집:** Rocky Linux 환경에서 `rsyslog`와 `MariaDB`를 유기적으로 연동하여, 흩어져 있는 시스템 로그를 한 곳으로 자동 수집하는 환경을 구축했습니다.
* **실시간 모니터링:** `LogAnalyzer`를 활용하여 수집된 로그 데이터를 웹 기반 실시간 모니터링 대시보드 인터페이스로 시각화했습니다.

### 2. 이기종 OS 기반 통합 인프라 서비스 운영
* **Windows 웹 서비스:** Windows Server의 IIS를 이용하여 기업형 웹 서비스 대문 페이지를 제작하고 호스팅했습니다.
* **Linux 파일 서버 연동:** 리눅스 환경에 `vsftpd` 서버를 구축하고, 가상머신 간 FTP 프로토콜을 이용하여 서로 다른 OS 간에 파일을 원활하게 송수신하는 실무 실습을 진행했습니다.

---

## 💭 오늘의 회고

### 배운 점
* 단순한 단일 서비스 설치를 넘어, **Linux와 Windows라는 서로 다른 OS(이기종 환경)가 네트워크를 통해 어떻게 상호작용하고 데이터를 주고받는지** 그 거시적인 흐름을 체득했습니다.
* 서비스 간의 권한(DB User 설정)과 OS 레벨의 보안 정책(SELinux, Firewall)이 실제 통신과 인프라 운영에 얼마나 결정적인 영향을 미치는지 깊이 있게 이해하게 되었습니다.

### 트러블슈팅 (어려운 점 및 극복 과정)
* **복합 장애 해결:** 이번 실습 과제가 요구하는 기술적 범위가 워낙 넓어 체력적으로나 정신적으로 큰 한계를 느꼈습니다. 특히 중앙 로그 서버 구축 시 발생한 'DB 권한 에러'와, 웹 서버 연동 시 발생한 '포트 및 파일 확장자 인식 문제'가 동시에 겹쳐 발생했을 때 원인을 특정하기가 매우 까다로웠습니다.
* **성장 포인트:** 수많은 에러 로그 속에서 해결의 실마리를 찾기란 고통스러운 과정이었지만, 네트워크 단절 및 서비스 거부의 원인을 계층별(네트워크, 방화벽, 애플리케이션 권한)로 쪼개어 분석하는 인내심과 디버깅 시야를 넓힐 수 있었습니다.
